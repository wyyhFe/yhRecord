# 打卡模块 V2 设计文档

> 设计日期：2026-06-15
> 设计目标：从"记录工具"升级为"习惯养成引擎"

## 核心理念

打卡最核心的驱动力是**成就感**和**连续性**。如果仅仅是数据库里多了一条记录，用户很快就会失去动力。

V2 围绕四个维度升级：

```
┌─────────────────────────────────────────────────────┐
│                   打卡 V2 架构                       │
├──────────────┬──────────────┬──────────────┬────────┤
│  成就驱动    │  内容丰富    │  跨模块联动  │  防流失 │
│              │              │              │        │
│ • 热力图日历  │ • 心情 Emoji  │ • 打卡→日记   │ • 补卡机制│
│ • 勋章系统    │ • 预设标签    │ • 记账→打卡   │ • 断签保护│
│ • 连续天数    │ • 50字微记录  │              │        │
│ • 里程碑      │ • 图片(已有)  │              │        │
└──────────────┴──────────────┴──────────────┴────────┘
```

## 一、热力图日历

用颜色深浅直观展示打卡频率，连续打卡形成漂亮色块，断签时产生强烈的"损失厌恶"心理。

### 色阶规则

| 完成比例 | 颜色 | 语义 |
|----------|------|------|
| 0% | 浅灰 | 未打卡 |
| 1%-49% | 浅紫 | 部分完成 |
| 50%-99% | 中紫 | 大部分完成 |
| 100% | 深紫 | 全部完成 |

### 接口

```
GET /checkin/heatmap?year=2026&month=6
```

返回：每天的完成数、当前连续天数、历史最佳连续天数、本月打卡天数。

### 连续天数算法

从今天（或昨天）倒推，逐天检查 `biz_checkin_record` 是否有记录，遇断签停止。支持补卡场景。

## 二、勋章与成就系统

里程碑奖励驱动长期坚持。勋章不只是"收集品"，而是用户行为的叙事线索。

### 勋章分类

| 分类 | 图标 | 解锁逻辑 |
|------|------|----------|
| 连续坚持 STREAK | 🔥 | 连续打卡 N 天 |
| 累计总量 TOTAL | 📊 | 总打卡次数达到 N |
| 时间习惯 TIME | ⏰ | 特定时间段打卡 |
| 特殊成就 SPECIAL | ⭐ | 复合条件 |

### 预设勋章（12 枚）

| 勋章 | 条件 | 难度 |
|------|------|------|
| 🔥 初心者 | 连续 3 天 | ⭐ |
| 🔥 小坚持 | 连续 7 天 | ⭐⭐ |
| 🔥 习惯养成 | 连续 21 天 | ⭐⭐⭐ |
| 🔥 自律达人 | 连续 30 天 | ⭐⭐⭐⭐ |
| 🔥 钢铁意志 | 连续 100 天 | ⭐⭐⭐⭐⭐ |
| 📊 百次打卡 | 累计 100 次 | ⭐⭐⭐ |
| 📊 千次打卡 | 累计 1000 次 | ⭐⭐⭐⭐⭐ |
| ⏰ 早起鸟 | 6:00 前累计 7 次 | ⭐⭐⭐ |
| ⏰ 夜猫子 | 23:00 后累计 7 次 | ⭐⭐ |
| ⭐ 全勤日 | 一天完成所有任务 | ⭐⭐ |
| ⭐ 多面手 | 同时 5 个以上任务 | ⭐⭐ |
| ⭐ 回归者 | 断签后重新连续 7 天 | ⭐⭐⭐ |

### 判定机制

打卡成功后自动检查所有未解锁勋章，逐个判定。判定通过 → 插入 `biz_user_medal` → 前端弹窗庆祝。

### 接口

```
GET /checkin/medals    — 获取勋章列表（含解锁状态）
```

## 三、心情 Emoji + 预设标签

打卡时多一点点仪式感。

### 心情 Emoji

10 个可选：😊 😐 😫 💪 🎉 😴 🤢 🔥 🌧️ ⭐

点击选中，再点取消，可不选。

### 预设标签

20 个系统标签，按场景分组：运动、早起、饮食、学习、睡眠、生活。

用户可创建自定义标签，打卡时最多选 3 个。

### 接口

```
GET  /checkin/tags    — 获取标签列表
POST /checkin/tags    — 创建自定义标签
```

## 四、打卡微记录

50 字极简记录，降低心理负担。复用 `remark` 字段，前端 50 字软限制。

---

## 五、跨模块联动（规划中）

### 打卡 → 日记

写日记时自动提取当天打卡汇总，插入日记顶部作为"今日回顾"素材。

### 记账 → 打卡

打卡任务可关联记账标签：
- 不消费类："每天不喝奶茶" → 有奶茶消费时自动失败
- 消费类："每天记账" → 有记账时自动成功
- 金额目标："每月存钱 2000" → 达标时自动成功

## 六、补卡机制

| 规则 | 说明 |
|------|------|
| 补卡次数 | 每月 2 次免费 |
| 补卡范围 | 最近 7 天 |
| 补卡限制 | 同一天同一任务只能补一次 |
| 补卡标记 | `is_mend = 1`，UI 有特殊标识 |

### 接口

```
POST /checkin/records/mend     — 补卡
GET  /checkin/mend-remaining   — 查询当月剩余补卡次数
```
