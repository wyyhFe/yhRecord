# 小程序订阅消息和地图配置步骤

本文档按“先跑通订阅消息，再跑通地图”的顺序整理。  
你可以照着一步一步做，不需要一次全做完。

---

## 一、先做什么

建议按这个顺序处理：

1. 配置小程序订阅消息模板
2. 配置前后端模板 ID
3. 配置模板字段映射
4. 真机测试订阅授权和消息发送
5. 配置腾讯地图 Key
6. 配置小程序合法域名
7. 真机测试地图定位和选点

---

## 二、小程序订阅消息配置

### 1. 你要准备哪些模板

当前项目建议先准备两个模板：

- 日记提醒
- 纪念日提醒

后续如果你要做记账提醒，再额外配置：

- 记账提醒

---

### 2. 去哪里配置模板

登录微信小程序后台后，找到订阅消息模板管理页面，新增模板。

你需要记录每个模板的两类信息：

1. 模板 ID
2. 模板字段名

注意：

- 模板 ID 不是模板编号
- 模板字段名决定后端发送时用什么 key

例如一个模板可能会长这样：

- `thing1`
- `time2`
- `thing3`

或者：

- `date1`
- `thing18`
- `amount10`
- `amount4`
- `amount5`

这两种都可能，取决于你选的模板。

---

### 3. 前端要配置什么

打开前端环境文件：

- [miniapp/.env.development](/e:/01-server/lifeRecord/miniapp/.env.development)
- [miniapp/.env.production](/e:/01-server/lifeRecord/miniapp/.env.production)

至少要填：

```env
VITE_MP_DIARY_TEMPLATE_ID=你的日记提醒模板ID
VITE_MP_MEMORIAL_TEMPLATE_ID=你的纪念日提醒模板ID
```

如果后面做记账提醒，再加：

```env
VITE_MP_LEDGER_TEMPLATE_ID=你的记账提醒模板ID
```

前端当前会在提醒设置页唤起订阅授权，页面文件是：

- [reminder.vue](/e:/01-server/lifeRecord/miniapp/src/pages/profile/reminder.vue)

---

### 4. 后端要配置什么

打开后端配置文件：

- [application.yaml](/e:/01-server/lifeRecord/record/src/main/resources/application.yaml)

先配置模板 ID：

```yaml
app:
  reminder:
    mini-program:
      diary-template-id: 你的日记提醒模板ID
      memorial-template-id: 你的纪念日提醒模板ID
      ledger-template-id: 你的记账提醒模板ID
```

如果暂时没有记账提醒模板，可以先不改 `ledger-template-id`。

---

## 三、模板字段映射怎么配

### 1. 为什么要配字段映射

微信模板消息不是只配一个模板 ID 就完了。  
后端发送时，`data` 里的字段名必须和你模板里的字段名一一对应。

如果模板字段不一致，微信会拒绝发送。

现在项目已经支持把字段名配到 `yaml`，不用以后每换模板都改 Java 代码。

相关配置类：

- [AppProperties.java](/e:/01-server/lifeRecord/record/src/main/java/com/record/common/config/AppProperties.java)

相关发送逻辑：

- [ReminderServiceImpl.java](/e:/01-server/lifeRecord/record/src/main/java/com/record/modules/reminder/service/impl/ReminderServiceImpl.java)

---

### 2. 日记提醒字段映射

如果你的日记提醒模板字段是：

- 标题字段：`thing1`
- 时间字段：`time2`
- 内容字段：`thing3`

那默认就能直接用：

```yaml
app:
  reminder:
    mini-program:
      diary-fields:
        title-field: thing1
        time-field: time2
        content-field: thing3
```

如果你模板字段不一样，就改成自己的。

---

### 3. 纪念日提醒字段映射

如果你的纪念日模板字段是：

- 标题字段：`thing1`
- 日期字段：`date2`
- 内容字段：`thing3`

那配置是：

```yaml
app:
  reminder:
    mini-program:
      memorial-fields:
        title-field: thing1
        date-field: date2
        content-field: thing3
```

---

### 4. 记账提醒字段映射

如果你后面要接“账单通知”模板：

- `date1`
- `thing18`
- `amount10`
- `amount4`
- `amount5`

那配置就是：

```yaml
app:
  reminder:
    mini-program:
      ledger-fields:
        period-field: date1
        type-field: thing18
        amount-field: amount10
        expense-field: amount4
        income-field: amount5
```

注意：

- 现在项目已经支持“字段映射配置化”
- 但“记账提醒调度发送”还没完整接入主链
- 也就是说配置先能放进去，但真正定时发记账提醒还需要继续开发

## 四、怎么测试小程序订阅消息

### 1. 前端测试

进入提醒设置页：

- [reminder.vue](/e:/01-server/lifeRecord/miniapp/src/pages/profile/reminder.vue)

测试步骤：

1. 打开“小程序提醒”开关
2. 微信弹订阅授权框
3. 点击“允许”
4. 点击“保存提醒设置”

如果成功，前端应该能保存开关状态。

---

### 2. 后端测试

后端相关接口：

- 查询提醒设置：`GET /reminders/settings`
- 保存提醒设置：`PUT /reminders/settings`

你可以用 Knife4j 或 Apifox 看返回值。

调度任务文件：

- [ReminderScheduler.java](/e:/01-server/lifeRecord/record/src/main/java/com/record/scheduler/ReminderScheduler.java)

当前已接入：

- 日记提醒调度
- 纪念日提醒调度

---

### 3. 真机测试重点

你要确认：

- 能否弹出订阅授权框
- 用户允许后是否能保存成功
- 到了设定时间后，是否能收到微信服务通知

注意：

- 小程序订阅消息不是发到普通聊天对话框
- 它通常进入微信“服务通知”

## 五、地图功能配置

### 1. 现在项目里的地图链路

当前地图不是只靠前端，也不是只靠后端，而是前后端配合：

前端：

- 获取当前位置
- 手动选点
- 通过 `qqmap` 工具层调用腾讯地图 SDK

后端：

- 调腾讯地图 `WebService API`
- 把经纬度补全成结构化地址

前端相关目录：

- [miniapp/src/utils/qqmap](/e:/01-server/lifeRecord/miniapp/src/utils/qqmap)

位置组件：

- [LocationPicker](e:/01-server/lifeRecord/miniapp/src/components/business/location-picker/index.vue)

后端逆地理编码接口：

- `POST /locations/reverse-geocode`

---

### 2. 腾讯地图后台要做什么

登录腾讯位置服务后台后：

1. 找到你的 Key
2. 确认这个 Key 开启了 `WebServiceAPI`

因为后端逆地理编码依赖这个能力。

你当前的腾讯地图 Key 是：

```text
DWUBZ-T4VKJ-7XCFD-XLNNS-SLNVF-JKBFJ
```

后端配置位置：

- [application.yaml](/e:/01-server/lifeRecord/record/src/main/resources/application.yaml)

配置项：

```yaml
app:
  map:
    api-key: 你的腾讯地图key
```

---

### 3. 微信小程序后台要做什么

登录微信小程序后台，进入开发设置页面，配置合法域名。

至少加：

```text
https://apis.map.qq.com
```

如果你的项目还会访问：

- 自己的后端域名
- OSS 域名

也要一起加进去。

---

### 4. 地图功能测试步骤

进入写日记页：

- [editor.vue](/e:/01-server/lifeRecord/miniapp/src/pages/diary/editor.vue)

测试这几项：

1. 点击“当前位置”
2. 看是否能拿到经纬度
3. 看是否能补出 `address/province/city/district`
4. 点击“手动选择”
5. 看是否能完成选点并回填位置

如果你要测试后端接口本身：

- 调 `POST /locations/reverse-geocode`

看返回是否包含：

- `locationName`
- `address`
- `province`
- `city`
- `district`
- `latitude`
- `longitude`

## 七、你现在应该先做什么

建议按这个顺序：

1. 去小程序后台配置“日记提醒模板”
2. 去小程序后台配置“纪念日提醒模板”
3. 记下这两个模板的字段名
4. 把模板 ID 写入前后端配置

6. 真机测试订阅授权
7. 真机测试日记提醒发送
8. 再配置腾讯地图 Key 和合法域名
9. 测试当前位置和手动选点
