## 2. 当前已支持接口

- `POST /knowledge/bases/create`
  - 创建知识库
- `GET /knowledge/bases/list`
  - 查询当前用户知识库列表
- `POST /knowledge/documents/create`
  - 创建知识库文档元数据，并自动创建一条 `PARSE` 任务
- `GET /knowledge/documents/list?knowledgeBaseId=1`
  - 查询指定知识库下的文档列表
- `GET /knowledge/tasks/list?knowledgeBaseId=1`
  - 查询指定知识库下的任务列表
- `GET /knowledge/tasks/list?knowledgeBaseId=1&documentId=1`
  - 查询指定文档的任务列表

## 3. 当前实现边界

- 已完成 `MySQL` 侧元数据管理
- 已完成知识库、文档、任务三张表对应的后端骨架
- 已完成用户维度的数据隔离校验
- 已完成基础字段校验：
  - `visibility` 仅支持 `PRIVATE / PUBLIC`
  - `sourceType` 仅支持 `UPLOAD / URL / MANUAL`
- 创建文档时会自动写入一条 `PARSE` 任务
