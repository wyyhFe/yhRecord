# 生活记录小程序新手发布步骤

这份文档按“先本地跑通，再准备微信后台配置，最后提交发布”的顺序写。你照着做就行。

## 第 1 步：准备账号和资料
你先准备好下面这些东西：

1. 微信小程序账号
2. 小程序 `AppID`
3. 腾讯位置服务 `WebService Key`
4. 阿里云 OSS bucket
5. 一台可部署后端的服务器
6. 一个后端 HTTPS 域名
7. 一个 OSS 或 CDN 访问域名

如果你后面还要做扩展提醒，再额外准备：

1. 公众号 `AppID`
2. 公众号 `AppSecret`
3. 公众号模板消息模板 ID

## 第 2 步：本地跑通项目

### 2.1 启动后端
你本地需要先装好：

- JDK 17+
- MySQL
- Redis

然后配置开发环境：

- 文件：[application-dev.yaml](e:/01-server/lifeRecord/record/src/main/resources/application-dev.yaml)

启动后端：

```powershell
cd e:\01-server\lifeRecord\record
.\mvnw.cmd spring-boot:run
```

### 2.2 启动前端
文件：

- [.env.development](e:/01-server/lifeRecord/miniapp/.env.development)

确认本地接口地址：

- `VITE_API_BASE_URL=http://127.0.0.1:8080`

启动前端调试：

```powershell
cd e:\01-server\lifeRecord\miniapp
npm install
npm run dev:mp-weixin
```

### 2.3 本地验证
你至少验证下面几件事：

1. 小程序能打开首页
2. 可以微信登录
3. 可以创建日记
4. 可以上传图片到 OSS
5. 可以获取当前位置或手动选点

## 第 3 步：填写小程序基础配置

### 3.1 修改小程序 AppID
文件：

- [manifest.json](e:/01-server/lifeRecord/miniapp/src/manifest.json)

把：

- `mp-weixin.appid`

改成你自己的小程序 `AppID`。

### 3.2 配置微信后台合法域名
进入微信小程序后台，配置：

1. `request` 合法域名
2. `uploadFile` 合法域名
3. `downloadFile` 合法域名

一般对应：

1. 你的后端域名
2. 你的 OSS 域名
3. 你的 OSS 或 CDN 域名

## 第 4 步：配置地图能力
你当前后端的定位补全地址，依赖腾讯位置服务。

要做的事：

1. 去腾讯位置服务后台创建应用
2. 开通 `WebService API`
3. 拿到 `Key`

然后把它配置到后端：

- 文件：[application.yaml](e:/01-server/lifeRecord/record/src/main/resources/application.yaml)
- 配置项：`app.map.api-key`

生产环境建议走环境变量：

- `TENCENT_MAP_API_KEY`

## 第 5 步：配置微信登录
你现在项目用的是小程序 OpenID 登录。

要做的事：

1. 拿到小程序 `AppID`
2. 拿到小程序 `AppSecret`

配置方式：

- 文件：[application.yaml](e:/01-server/lifeRecord/record/src/main/resources/application.yaml)
- 配置项：
  - `app.wechat.app-id`
  - `app.wechat.app-secret`

生产环境建议走环境变量：

- `WECHAT_APP_ID`
- `WECHAT_APP_SECRET`

## 第 6 步：配置 OSS
你现在是前端直传 OSS，后端只生成签名和保存路径。

要做的事：

1. 准备 OSS bucket
2. 准备 `AccessKeyId`
3. 准备 `AccessKeySecret`
4. 准备正式访问域名

后端配置：

- `ALIOSS_ENDPOINT`
- `ALIOSS_BUCKET_NAME`
- `ALIOSS_ACCESS_KEY_ID`
- `ALIOSS_ACCESS_KEY_SECRET`

前端配置：

- 文件：[.env.production](e:/01-server/lifeRecord/miniapp/.env.production)
- 配置项：`VITE_OSS_BASE_URL`

## 第 7 步：配置提醒
提醒建议按两层做：

### 7.1 主提醒：小程序订阅消息
这是你首发应该先做的。

操作顺序：

1. 在微信小程序后台创建订阅消息模板
2. 至少准备两个模板：
   - 日记提醒模板
   - 纪念日提醒模板
3. 记录模板 ID
4. 配到后端：
   - `MP_DIARY_TEMPLATE_ID`
   - `MP_MEMORIAL_TEMPLATE_ID`

说明：

- 小程序订阅消息通常进入“服务通知”
- 不是普通微信聊天对话框
- 用户必须在前端先同意订阅

### 7.2 扩展提醒：公众号模板消息
这不是首发必须，但你可以提前配。

操作顺序：

1. 准备公众号
2. 获取公众号 `AppID` 和 `AppSecret`
3. 配置公众号模板消息模板
4. 记录模板 ID
5. 配到后端：
   - `OA_APP_ID`
   - `OA_APP_SECRET`
   - `OA_DIARY_TEMPLATE_ID`
   - `OA_MEMORIAL_TEMPLATE_ID`

注意：

- 公众号提醒还需要做用户 `openid` 绑定
- 小程序 `openid` 和公众号 `openid` 不是一个值

## 第 8 步：配置生产环境后端
建议你服务器上用环境变量启动，不要把密钥硬写进文件。

你至少要准备这些环境变量：

```env
SPRING_PROFILES_ACTIVE=prod
DB_URL=jdbc:mysql://...
DB_USERNAME=...
DB_PASSWORD=...
REDIS_HOST=...
REDIS_PORT=6379
REDIS_DATABASE=0
APP_JWT_SECRET=...
WECHAT_APP_ID=...
WECHAT_APP_SECRET=...
TENCENT_MAP_API_KEY=...
ALIOSS_ENDPOINT=oss-cn-hangzhou.aliyuncs.com
ALIOSS_BUCKET_NAME=...
ALIOSS_ACCESS_KEY_ID=...
ALIOSS_ACCESS_KEY_SECRET=...
MP_DIARY_TEMPLATE_ID=...
MP_MEMORIAL_TEMPLATE_ID=...
OA_APP_ID=...
OA_APP_SECRET=...
OA_DIARY_TEMPLATE_ID=...
OA_MEMORIAL_TEMPLATE_ID=...
```

## 第 9 步：构建小程序
前端生产环境文件：

- [.env.production](e:/01-server/lifeRecord/miniapp/.env.production)

至少确认：

- `VITE_API_BASE_URL` 是线上后端 HTTPS 域名
- `VITE_OSS_BASE_URL` 是线上 OSS/CDN 域名

构建命令：

```powershell
cd e:\01-server\lifeRecord\miniapp
npm run build:mp-weixin
```

构建产物目录：

- [dist/build/mp-weixin](e:/01-server/lifeRecord/miniapp/dist/build/mp-weixin)

## 第 10 步：导入微信开发者工具
打开微信开发者工具：

1. 选择“导入项目”
2. 目录选择：
   - [dist/build/mp-weixin](e:/01-server/lifeRecord/miniapp/dist/build/mp-weixin)
3. 确认 AppID 正确
4. 编译项目

然后重点测试：

1. 登录
2. 日记创建
3. 图片上传
4. 定位
5. 记账
6. 打卡
7. 提醒设置页

## 第 11 步：提交审核前检查
你提交审核前，至少确认：

1. 后端域名可访问
2. OSS 域名可访问
3. 小程序合法域名已配置
4. 微信登录可用
5. 地图接口可用
6. 订阅消息模板已配置
7. 线上数据库和 Redis 正常
8. 敏感密钥没有写死在仓库里

## 第 12 步：提交发布
最后一步就是在微信开发者工具里：

1. 上传代码
2. 填版本号和项目备注
3. 到微信小程序后台提交审核
4. 审核通过后发布

## 你现在最优先要做的三件事
如果你不知道先做什么，就先按这个顺序：

1. 把 `manifest.json` 的小程序 `appid` 换成你自己的
2. 把腾讯地图 key、微信 `app-id/app-secret`、OSS 参数配好
3. 优先配置“小程序订阅消息模板”，公众号消息先留作扩展
