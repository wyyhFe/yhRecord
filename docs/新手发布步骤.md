# 新手发布步骤

这份文档按“先本地跑通，再配置微信后台，最后构建发布”的顺序整理。你照着一步一步做即可。

## 第 1 步：准备账号和基础资源
先准备好下面这些内容：

1. 微信小程序账号
2. 小程序 `AppID`
3. 小程序 `AppSecret`
4. 腾讯位置服务 `Key`
5. 阿里云 OSS bucket
6. 一台可部署后端的服务器
7. 一个后端 HTTPS 域名
8. 一个 OSS 或 CDN 访问域名

如果后面还要做公众号扩展提醒，再额外准备：

1. 公众号 `AppID`
2. 公众号 `AppSecret`
3. 公众号模板消息模板 ID

## 第 2 步：本地跑通项目

### 2.1 启动后端
本地先装好：

- JDK 17+
- MySQL
- Redis

开发环境配置文件：

- [application-dev.yaml](e:/01-server/lifeRecord/record/src/main/resources/application-dev.yaml)

启动后端：

```powershell
cd e:\01-server\lifeRecord\record
.\mvnw.cmd spring-boot:run
```

### 2.2 启动前端
前端开发环境文件：

- [miniapp/.env.development](e:/01-server/lifeRecord/miniapp/.env.development)

确认本地接口地址：

- `VITE_API_BASE_URL=http://127.0.0.1:8080`

启动前端调试：

```powershell
cd e:\01-server\lifeRecord\miniapp
npm install
npm run dev:mp-weixin
```

### 2.3 本地验证
至少验证以下功能：

1. 小程序首页能正常打开
2. 能进行微信登录
3. 能创建日记
4. 能上传图片到 OSS
5. 能获取当前位置或手动选点
6. 能进入提醒设置页

## 第 3 步：填写小程序基础配置

### 3.1 修改小程序 AppID
文件：

- [manifest.json](e:/01-server/lifeRecord/miniapp/src/manifest.json)

把：

- `mp-weixin.appid`

改成你自己的小程序 `AppID`。当前已填入：

- `wxc881594b9650474f`

### 3.2 配置微信后台合法域名
登录微信小程序后台，配置：

1. `request` 合法域名
2. `uploadFile` 合法域名
3. `downloadFile` 合法域名

一般对应：

1. 你的后端域名
2. 你的 OSS 域名
3. 你的 OSS 或 CDN 域名

## 第 4 步：配置地图能力
当前项目的地图能力是“前端 + 后端共用同一个腾讯位置服务 Key”的模式。

### 4.1 这个 Key 用在哪些地方
同一个 Key 可以同时用于：

1. 前端小程序 `qqmap-wx-jssdk`
2. 后端 `WebService API` 逆地理编码

你当前的 Key：

- `DWUBZ-T4VKJ-7XCFD-XLNNS-SLNVF-JKBFJ`

### 4.2 你要做的配置
去腾讯位置服务后台：

1. 找到这个 Key
2. 确认已经开启 `WebServiceAPI`
3. 如果控制台里有“引用管理”，确保允许你当前小程序使用

后端配置位置：

- [application.yaml](e:/01-server/lifeRecord/record/src/main/resources/application.yaml)
- 配置项：`app.map.api-key`

前端配置位置：

- [miniapp/.env.development](e:/01-server/lifeRecord/miniapp/.env.development)
- [miniapp/.env.production](e:/01-server/lifeRecord/miniapp/.env.production)
- 配置项：`VITE_TENCENT_MAP_KEY`

### 4.3 个人开发者能不能用
可以。

你作为个人开发者可以使用腾讯位置服务，只要控制台给你的 Key 开通了对应能力且有配额即可。
你在“配额管理”里看到的每日次数，就是这个 Key 当前可用额度。
前期个人开发做当前位置、手动选点、逆地理编码，通常够用。

### 4.4 微信后台还要加什么域名
微信小程序后台至少加：

- `https://apis.map.qq.com`

如果项目还会访问你自己的后端和 OSS，也一起加入合法域名。

## 第 5 步：配置微信登录
当前项目使用的是“小程序 OpenID 登录”。

后端配置文件：

- [application.yaml](e:/01-server/lifeRecord/record/src/main/resources/application.yaml)

需要确认：

- `app.wechat.app-id = wxc881594b9650474f`
- `app.wechat.app-secret = dc5e4819af366f49a71214eda8b28015`

生产环境建议改成环境变量：

- `WECHAT_APP_ID`
- `WECHAT_APP_SECRET`

## 第 6 步：配置 OSS
当前是前端直传 OSS，后端负责签名和保存路径。

你要准备：

1. OSS bucket
2. `AccessKeyId`
3. `AccessKeySecret`
4. 正式访问域名

后端环境变量：

- `ALIOSS_ENDPOINT`
- `ALIOSS_BUCKET_NAME`
- `ALIOSS_ACCESS_KEY_ID`
- `ALIOSS_ACCESS_KEY_SECRET`

前端生产环境：

- [miniapp/.env.production](e:/01-server/lifeRecord/miniapp/.env.production)
- 配置项：`VITE_OSS_BASE_URL`

## 第 7 步：配置提醒
当前提醒方案分两层：

1. 主提醒：小程序订阅消息
2. 扩展提醒：公众号模板消息

### 7.1 小程序订阅消息
这是首发必须先配好的。

你现在已经配了 4 个模板：

1. 日记提醒
2. 纪念日提醒
3. 每日记账提醒
4. 记账月报通知

前端环境文件：

- [miniapp/.env.development](e:/01-server/lifeRecord/miniapp/.env.development)
- [miniapp/.env.production](e:/01-server/lifeRecord/miniapp/.env.production)

后端配置文件：

- [application.yaml](e:/01-server/lifeRecord/record/src/main/resources/application.yaml)

提醒页：

- [reminder.vue](e:/01-server/lifeRecord/miniapp/src/pages/profile/reminder.vue)

当前小程序提醒规则：

- 日记提醒：每天晚上 `22:00`，当天未写日记才提醒
- 每日记账提醒：每天晚上 `20:30`，当天未记账才提醒
- 记账月报提醒：每月最后一天晚上 `21:00` 发送
- 纪念日提醒：每天早上 `08:05` 检查并发送

### 7.2 公众号模板消息
这是扩展方案，不是首发必须项。

当前已填公众号配置：

- `AppID = wxe0b2c126bb08961c`
- `AppSecret = 7c7db12875ab6c4eafabf87d66bf6dd8`

但要真正使用公众号提醒，你后面还要补：

1. 用户公众号 `openid` 绑定流程
2. 公众号模板 ID 配置
3. 公众号消息发送联调

## 第 9 步：构建小程序
前端生产环境文件：

- [miniapp/.env.production](e:/01-server/lifeRecord/miniapp/.env.production)

确认：

- `VITE_API_BASE_URL` 是线上后端 HTTPS 域名
- `VITE_OSS_BASE_URL` 是线上 OSS 或 CDN 域名

构建命令：

```powershell
cd e:\01-server\lifeRecord\miniapp
npm run build:mp-weixin
```

构建产物目录：

- [dist/build/mp-weixin](e:/01-server/lifeRecord/miniapp/dist/build/mp-weixin)

## 第 10 步：导入微信开发者工具
打开微信开发者工具：

1. 选择“导入项目”
2. 目录选择：
   - [dist/build/mp-weixin](e:/01-server/lifeRecord/miniapp/dist/build/mp-weixin)
3. 确认 AppID 正确
4. 编译项目

重点测试：

1. 登录
2. 日记创建
3. 图片上传
4. 地图定位
5. 记账
6. 提醒设置
7. 小程序订阅消息授权

## 第 11 步：提交审核前检查
至少确认：

1. 后端域名可访问
2. OSS 域名可访问
3. 小程序合法域名已配置
4. 微信登录可用
5. 地图接口可用
6. 订阅消息模板已配置
7. 数据库和 Redis 正常
8. 敏感密钥没有继续硬编码到仓库

## 第 12 步：提交发布
最后在微信开发者工具中：

1. 上传代码
2. 填版本号和项目备注
3. 到微信小程序后台提交审核
4. 审核通过后发布

## 当前最优先要做的三件事
如果你现在不知道先做什么，就按这个顺序：

1. 把微信后台合法域名和地图域名配好
2. 真机测试小程序订阅消息授权和提醒发送
3. 再做一次完整的登录、日记、记账、地图联调
