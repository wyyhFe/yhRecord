# 生活记录小程序发布配置清单

## 1. 前端环境配置
前端已经拆成两套环境：

- 本地开发：[miniapp/.env.development](e:/01-server/lifeRecord/miniapp/.env.development)
- 生产环境：[miniapp/.env.production](e:/01-server/lifeRecord/miniapp/.env.production)

当前前端读取的环境变量有：

- `VITE_APP_NAME`
- `VITE_API_BASE_URL`
- `VITE_OSS_BASE_URL`

对应代码入口：

- [app.ts](e:/01-server/lifeRecord/miniapp/src/config/app.ts)
- [env.d.ts](e:/01-server/lifeRecord/miniapp/env.d.ts)

说明：

- 本地开发时，`VITE_API_BASE_URL` 指向本机 Spring Boot 地址，例如 `http://127.0.0.1:8080`
- 生产构建时，`VITE_API_BASE_URL` 必须替换成线上 HTTPS 域名
- `VITE_OSS_BASE_URL` 用于前端展示图片完整地址，必须和你正式访问 OSS 或 CDN 的域名一致

## 6. 提醒方案配置
当前建议采用两层方案：

- 主提醒：小程序订阅消息
- 扩展提醒：公众号模板消息

### 6.1 小程序订阅消息
优先配置以下内容：

- `app.reminder.mini-program.diary-template-id`
- `app.reminder.mini-program.memorial-template-id`

说明：

- 小程序订阅消息更适合个人开发者
- 消息通常进入微信“服务通知”入口
- 用户必须先在前端授权订阅，后端才能发送

### 6.2 公众号模板消息
如果你后续还要做扩展提醒，再配置：

- 公众号 `AppID`
- 公众号 `AppSecret`
- 日记提醒模板 ID
- 纪念日提醒模板 ID
- 用户绑定公众号 `openid` 的完整流程

## 7. OSS 配置
发布前要确认：

- OSS bucket 可用
- 小程序后台已配置 OSS 域名为合法上传/下载域名
- 生产环境前端 `VITE_OSS_BASE_URL` 已改成真实可访问域名

## 8. 微信后台域名配置
在微信小程序后台需要配置：

- `request` 合法域名：后端接口域名
- `uploadFile` 合法域名：OSS 域名
- `downloadFile` 合法域名：如果有下载能力，也要加
