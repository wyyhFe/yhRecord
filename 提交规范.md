# 提交规范

按”单次提交 + 多行分类说明”的方式提交本次改动：

1. `subject` 使用 1 行中文总标题。
2. `body` 按多行分类前缀编写，可用：`feat:` / `fix:` / `refactor:` / `perf:` / `docs:` / `chore:` / `test:`。
3. 每行只写 1 句，描述具体改动和结果，不要写空话。
4. 提交前先按模块归类改动，并先给我预览提交说明。
5. 我确认后，再执行 `git add`、`git commit`、`git push`。
6. 默认全部使用中文。
7. 整体不要超过 8 行。

## 推送节奏

- 不要每写完一个接口/文件就 push，应等**整个模块或一个完整功能点开发完毕**后再推送。
- 推送前必须先跑完编译验证（后端 `compile` + 前端 `type-check` + 前端 `build`），确认无报错后再 commit + push。
- 如果开发中途需要暂存进度，可以用 `git add` + `git commit` 但**不要 push**，等全部完成后再一次性 push。

## 文档补充
