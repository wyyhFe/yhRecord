## 2026-06-27 下午最终修复：Vditor 白屏 + 参考 grtblog 架构分析

### 问题根因
Vditor 渲染出来是白盒子，三个原因叠加：
1. `.editor-wrapper` 用了 `display: flex; flex-direction: column; overflow: hidden`，子元素 `.vditor-root` 用 `flex: 1; min-height: 0` —— 多层 flex 嵌套导致 Vditor 初始化时容器高度未解析（0 像素）
2. `:deep(.vditor) { height: 100% !important }` CSS 覆盖跟 Vditor 内部的 `height: h px` 配置冲突，导致 Vditor 内部高度计算塌陷
3. `onMounted` + `nextTick` 时 flex 布局计算未完成，`clientHeight` 读到 0

### 修复方案（参考 grtblog 的 absolute 定位模式）
- `.editor-wrapper`: `position: relative; flex: 1; min-height: 0` （去掉 flex-direction: column + overflow:hidden）
- `.vditor-root`: `position: absolute; inset: 0; border-radius: 8px; overflow: hidden` — 绝对定位填满容器，让 Vditor 原生的像素高度管理生效
- `:deep(.vditor)`: 只保留 `border: none !important; border-radius: 8px`，**去掉 `height: 100% !important`**
- 高度获取改用 **ResizeObserver**：监听 wrapper 的 `contentRect.height`，等到 > 0 时再调 `createVditor()`；500ms 兜底用 `clientHeight || 600`
- `height: h` 传给 Vditor 的 config，由 Vditor 内部管理；不再用 CSS 高度覆盖

### grtblog 对比分析
- grtblog 用的是 **CodeMirror 6** (`@codemirror/lang-markdown` + GFM)，**不是 Vditor**
- 渲染用 **markdown-it** + highlight.js（admin 端）/ **svmarkdown**（web 端 Svelte）
- 编辑器 height 靠 CSS 的 `height: 100%`，CodeMirror 原生支持；Vditor 需要显式 px 值
- 内容存储为原始 Markdown 文本（非 HTML），前后端渲染分离
- grtblog 的 `padding-bottom: 50vh` 思路很好：编辑器底部留白让光标永远不在视口最底部

### 当前状态
Vditor 回退后仍使用，不切 CodeMirror（保持现有技术栈）。核心修复是 CSS 布局从 flex 嵌套改为 absolute 定位 + ResizeObserver 延迟初始化。
